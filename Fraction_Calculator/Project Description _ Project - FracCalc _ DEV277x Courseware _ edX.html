You will be creating two classes, one called Fraction and the other called FractionCalculator. The Fraction class is an object that holds information about a fraction (numerator and denominator). It will have several constructors and both private and public methods implementing the behavior of a fraction. The FractionCalculator class is a class that will allow the user to enter in fractions and operations, calculating and displaying the result. It will run until the user tells it to quit. When this program is complete, you wont have to second guess your fraction arithmetic ever again!</p> <h2 id="part-1-fraction-class">Part 1 - Fraction Class</h2> <p>Create a new class called &quot;Fraction&quot; and include the following:</p> <h3 id="fields">Fields</h3> <ul> <li>two private instance variables to hold the numerator and denominator as ints</li> </ul> <h3 id="constructors">Constructors</h3> <ul> <li>a two parameter constructor that initializes the numerator and denominator <ul> <li>This constructor should throw an IllegalArgumentException if the denominator is zero</li> <li>If the user enters a negative denominator bump the negative sign to the numerator. For example, -3/-2 should be converted to 3/2. Likewise, 5/-3 should be converted to -5/3</li> <li><em>example</em>: Fraction myFrac = new Fraction(4,5); creates a fraction who's numerator is 4 and denominator is 5</li> </ul> </li> <li>one parameter constructor that initializes the object equal in value to the integer parameter. <ul> <li><em>example</em>: Fraction myFrac = new Fraction(3) would create a Fraction with numerator equal to 3 and denominator equal to 1.</li> </ul> </li> <li>zero parameter constructor that initializes the object to 0, meaning the numerator is 0 and the denominator is 1 <ul> <li><em>example</em>: Fraction myFrac = new Fraction(); would create a fraction who's decimal value is 0</li> </ul> </li> </ul> <p>You should eliminate as much redundancy as possible by letting your constructors rely on one another using the &quot;this&quot; keyword.</p> <h3 id="methods">Methods</h3> <table> <thead> <tr> <th>Method to implement</th> <th>parameter</th> <th>return</th> <th>description</th> </tr> </thead> <tbody> <tr> <td>getNumerator()</td> <td>none</td> <td>int</td> <td>exposes the value of the numerator field to the user</td> </tr> <tr> <td>getDenominator()</td> <td>none</td> <td>int</td> <td>exposes the value of the denominator field to the user</td> </tr> <tr> <td>toString()</td> <td>none</td> <td>String</td> <td>&quot;numerator/denominator&quot;, a String representation of the Fraction</td> </tr> <tr> <td>toDouble()</td> <td>none</td> <td>double</td> <td>the result of numerator / denominator</td> </tr> <tr> <td>add()</td> <td>Fraction other</td> <td>Fraction</td> <td>returns a new Fraction that is the sum of other and this fractions</td> </tr> <tr> <td>subtract()</td> <td>Fraction other</td> <td>Fraction</td> <td>returns a new Fraction that is the difference between the other and this fraction</td> </tr> <tr> <td>multiply()</td> <td>Fraction other</td> <td>Fraction</td> <td>returns a new Fraction that is the product of the other and this fraction</td> </tr> <tr> <td>divide()</td> <td>Fraction other</td> <td>Fraction</td> <td>returns a new Fraction that is the division of the other and this fraction, throw an IllegalArgumentException() if the user asks you to divide by 0</td> </tr> <tr> <td>equals()</td> <td>Object other</td> <td>boolean</td> <td>must take in an &quot;Object&quot; to properly override the Object class's equals method, but should ultimately check if two fractions are equal</td> </tr> <tr> <td>toLowestTerms()</td> <td>none</td> <td>none</td> <td>converts the current fraction to the lowest terms</td> </tr> <tr> <td>gcd()</td> <td>int num, int den</td> <td>int</td> <td>takes in two ints and determines the greatest common divisor of the two ints, should be a static method</td> </tr> </tbody> </table> <h4 id="equals">equals()</h4> <p>Override the Object equals() method so that it accurately determines whether or not two fractions are equal. In order to have it override, it has to take an Object as a parameter. Your method should check whether or not the parameter is an instanceof Fraction, since if it is not a Fraction it cannot be equal. Dont forget to cast the parameter to a Fraction after you check if it is an Object of type Fraction so that you can access its variables. Two fractions are equal if they represent the same number (i.e. 3/6 = 1/2 and -2/3 = 2/-3).</p> <h4 id="tolowestterms">toLowestTerms()</h4> <p>To convert a fraction to lowest terms we have to determine the greatest common divisor (factor) between the numerator and denominator. The greatest common divisor of two numbers a and b, is the largest number that evenly divides both a and b.</p> <p>The Euclidean Algorithm is a fast method for determining the GCD of two numbers. Here is pseudocode for its implementation:</p> <pre><code class="language-Java">while a and b are not zero find the remainder of a divided by b set a to b set b to the remainder you found return a </code></pre> <p>Here is an example of how it would work if a is 105 and b is 147. </p> <table> <thead> <tr> <th>Loop Iteration</th> <th>Value of a at end of loop run</th> <th>Value of b at end of loop run</th> <th>a % b</th> </tr> </thead> <tbody> <tr> <td>Pre-Loop</td> <td>105</td> <td>147</td> <td></td> </tr> <tr> <td>1</td> <td>147</td> <td>105</td> <td>105 % 147 = 105</td> </tr> <tr> <td>2</td> <td>105</td> <td>42</td> <td>147 % 105 = 147 - 105 = 42</td> </tr> <tr> <td>3</td> <td>42</td> <td>21</td> <td>105 % 42 = 105 - 84 = 21</td> </tr> <tr> <td>4</td> <td>21</td> <td>0</td> <td>42 % 21 = 0</td> </tr> </tbody> </table> <h4 id="gcd">gcd()</h4> <p>Implement gcd() as a public static method that takes two integers as parameters and returns an int that is their greatest common divisor.</p> <h2 id="part-2-fractioncalculator-class">Part 2 FractionCalculator Class</h2> <p>In this section, you will implement a FractionCalculator class that has a main method and three helper methods. Here is a screenshot from a sample run:</p> <p><img src="//prod-edxapp.edx-cdn.org/assets/courseware/v1/afafc9b3efb47b2c7f38e80ad77a69b5/asset-v1:Microsoft+DEV277x+1T2019a+type@asset+block/1-sampleOutput.png" alt="sample run" /></p> <p>Your program should be robust so that it if the user enters invalid input it will continue to re-prompt them until it is valid. Here is an example run where the user is confused and enters invalid input:</p> <p><img src="//prod-edxapp.edx-cdn.org/assets/courseware/v1/3d04cef31140243e34115b81cff6f500/asset-v1:Microsoft+DEV277x+1T2019a+type@asset+block/2-inputValidation.png" alt="input validation" /></p> <h3 id="methods-1">Methods</h3> <table> <thead> <tr> <th>Method to implement</th> <th>parameter</th> <th>return</th> <th>description</th> </tr> </thead> <tbody> <tr> <td>getOperation()</td> <td>Scanner input</td> <td>String</td> <td>Asks the user to enter in a valid mathematical operation. If a negative shows up anywhere else, then it is not a valid fraction. It may be helpful to remove the &quot;-&quot; character if there is one.</li> <li>If there is no &quot;/&quot; character, then every character in the string must be a number (if you removed the &quot;-&quot; sign).</li> <li>If there is a &quot;/&quot; character, then it may be helpful to create substrings for the numerator and denominator. <ul> <li>Both substrings must be non-empty.</li> <li>Both must be entirely made of numbers.</li> <li>The denominator cannot be &quot;0&quot;.</li> </ul> </li> </ul> <p><strong>Hint 1</strong>: It may be useful to create a helper method isNumber() that takes a String as input and returns true if every character in the String is a number 0-9 and false otherwise. This method can also check for empty strings. <strong>Hint 2</strong>: Once you determine whether or not the Strings are numbers, you may find the Integer.parseInt() method helpful.</p> <h4 id="getfraction">getFraction()</h4> <p>Here is example output from a call to <strong>getFraction()</strong>. No user input should throw an exception! If you are getting exceptions, then it is likely your validFraction method isnt correct.</p> <h2 id="part-3-putting-it-all-together">Part 3 - Putting it all together!</h2> <ol> <li>Write a short introduction method that describes the calculator program and welcomes your user</li> <li>Ask the user to enter in an operation</li> <li>As long as the user enters something thats not &quot;q&quot; or &quot;Q&quot; when asked for an operation you should run the calculator</li> <li>Get two fractions from the user and then perform whichever operation they ask for</li> <li>Print the result of the operation</li> </ol> <p>Here is an example run of the entire program: <img src="//prod-edxapp.edx-cdn.org/assets/courseware/v1/afafc9b3efb47b2c7f38e80ad77a69b5/asset-v1:Microsoft+DEV277x+1T2019a+type@asset+block/4-sampleRun.png" alt="full program run" /></p> <h2 id="part-4-hacker-problem-fractioncalculatoradvanced">Part 4 - Hacker Problem - FractionCalculatorAdvanced</h2> <p>Create another class called FractionCalculatorAdvanced. You may cut and paste useful methods from FractionCalculator. The key difference between FractionCalculator and FractionCalculatorAdvanced is that the user can enter in their operations on a single line. Allow the user to enter their input onto a single line. Your program must be robust so that if the user enters in invalid input, it will re-prompt them until they either enter a q to quit or a valid operation. It is possible to do this without try/catch, but it is quite difficult. Project Description

Module 2 Project - Fraction Calculator

This project is designed to help you practice building your own object class and testing it with a client class. You will be creating two classes, one called Fraction and the other called FractionCalculator. The Fraction class is an object that holds information about a fraction (numerator and denominator). It will have several constructors and both private and public methods implementing the behavior of a fraction. The FractionCalculator class is a class that will allow the user to enter in fractions and operations, calculating and displaying the result. It will run until the user tells it to quit. When this program is complete, you wont have to second guess your fraction arithmetic ever again!

Part 1 - Fraction Class

Create a new class called "Fraction" and include the following:

Fields

  • two private instance variables to hold the numerator and denominator as ints

Constructors

  • a two parameter constructor that initializes the numerator and denominator
    • This constructor should throw an IllegalArgumentException if the denominator is zero
    • If the user enters a negative denominator bump the negative sign to the numerator. For example, -3/-2 should be converted to 3/2. Likewise, 5/-3 should be converted to -5/3
    • example: Fraction myFrac = new Fraction(4,5); creates a fraction who's numerator is 4 and denominator is 5
  • one parameter constructor that initializes the object equal in value to the integer parameter.
    • example: Fraction myFrac = new Fraction(3) would create a Fraction with numerator equal to 3 and denominator equal to 1.
  • zero parameter constructor that initializes the object to 0, meaning the numerator is 0 and the denominator is 1
    • example: Fraction myFrac = new Fraction(); would create a fraction who's decimal value is 0

You should eliminate as much redundancy as possible by letting your constructors rely on one another using the "this" keyword.

equals()

Override the Object equals() method so that it accurately determines whether or not two fractions are equal. In order to have it override, it has to take an Object as a parameter. Your method should check whether or not the parameter is an instanceof Fraction, since if it is not a Fraction it cannot be equal. Dont forget to cast the parameter to a Fraction after you check if it is an Object of type Fraction so that you can access its variables. Two fractions are equal if they represent the same number (i.e. 3/6 = 1/2 and -2/3 = 2/-3).

toLowestTerms()

To convert a fraction to lowest terms we have to determine the greatest common divisor (factor) between the numerator and denominator. The greatest common divisor of two numbers a and b, is the largest number that evenly divides both a and b.

The Euclidean Algorithm is a fast method for determining the GCD of two numbers. Here is pseudocode for its implementation:

while a and b are not zero
    find the remainder of a divided by b
    set a to b
    set b to the remainder you found
return a

Here is an example of how it would work if a is 105 and b is 147.

Loop Iteration Value of a at end of loop run Value of b at end of loop run a % b
Pre-Loop 105 147
1 147 105 105 % 147 = 105
2 105 42 147 % 105 = 147 - 105 = 42
3 42 21 105 % 42 = 105 - 84 = 21
4 21 0 42 % 21 = 0

gcd()

Implement gcd() as a public static method that takes two integers as parameters and returns an int that is their greatest common divisor.

Some things to be mindful of when implementing the validFraction() method:

  • The first character may or may not be a "-" character. If a negative shows up anywhere else, then it is not a valid fraction. It may be helpful to remove the "-" character if there is one.
